# Domain Docs

This is a single-context repository.

## Before working in the codebase

- Read `CONTEXT.md` at the repository root
- Read relevant architectural decisions under `docs/adr/`
- Use the canonical vocabulary from the glossary in issues, plans, tests, and implementation discussions
- Surface conflicts with existing ADRs instead of silently overriding them
- Missing domain documents should not block work; create them lazily through domain modeling when terms or decisions are resolved
